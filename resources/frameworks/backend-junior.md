# Backend Júnior

## Visão Geral

Desenvolve funcionalidades simples a moderadas sob orientação direta de um Pleno ou Tech Lead. Foco principal em aprendizado, aderência a padrões e qualidade básica de entrega.

---

## Responsabilidades

- Implementar funcionalidades simples e moderadas conforme especificação técnica fornecida
- Escrever testes unitários básicos para o código produzido
- Seguir padrões, guias de estilo e convenções definidas pelo time
- Comunicar bloqueios com antecedência antes que afetem prazos
- Participar de code reviews como autor e, progressivamente, como revisor
- Manter qualidade mínima: logs adequados, tratamento de erros básico, sem quebra de testes existentes

---

## Não Responsabilidades

- Definir arquitetura de sistemas ou módulos
- Tomar decisões técnicas de alto impacto sem supervisão
- Liderar iniciativas ou projetos técnicos
- Mentorear outros membros do time
- Avaliar trade-offs tecnológicos estratégicos

---

## Interfaces

| Com quem | Natureza da interface |
|---|---|
| Backend Pleno / Sênior | Recebe mentoria técnica, direcionamento de implementação e code review |
| Tech Lead | Recebe contexto arquitetural e clareza sobre decisões técnicas do time |
| QA Engineer | Colabora na correção de bugs e entendimento de critérios de aceite |

---

## Competências Esperadas

- Fundamentos da linguagem principal do time (sintaxe, tipos, controle de fluxo, OO ou FP básico)
- Framework backend principal do time (estrutura de projeto, rotas, middlewares)
- Clean code básico: nomes claros, funções curtas, sem duplicação óbvia
- Git e versionamento (branches, commits, PRs)
- Testes unitários básicos
- Capacidade de aprendizado rápido e abertura a feedback
