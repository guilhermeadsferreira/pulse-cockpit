Criar `src/main/external/MetricsWriter.ts` com a classe `MetricsWriter`.

**Construtor:** recebe `workspacePath: string`, calcula `pessoasDir = join(workspacePath, 'pessoas')`.

**Metodo `writeAlerts(slug, alerts)`** (per D-01, D-11):
- `alerts: Array<{ tipo: 'blocker' | 'wip_alto' | 'cycle_time'; descricao: string; desde: string }>`.
- Substitui a secao `## Alertas Ativos` inteira (alerts ativos sao sempre o estado atual, nao append).
- Se `alerts` vazio, escreve `## Alertas Ativos\n\nNenhum alerta ativo.\n`.
- Cada alerta: `- **{tipo}**: {descricao} (desde {desde})`.

**Metodo `writeWeekly(slug, entry)`** (per D-02, D-09):
- `entry: { semana: string; velocity: number; deltaSP: number | null; prsMerged: number; reviews: number; collaborationScore: number; cycleTimeMedio: number }`.
- Append na secao `## Semanas` no formato:
  ```
  ### Semana {semana}
  - velocity: {velocity} SP {deltaStr}
  - PRs merged: {prsMerged}
  - reviews: {reviews}
  - collaboration: {collaborationScore}
  - cycle time médio: {cycleTimeMedio}d
  ```
- `deltaStr` = `(+N vs semana anterior)` ou `(-N vs semana anterior)` ou `(=)` quando `deltaSP` e 0 ou null mostra `(sem comparação)`.
- Aplica retencao: manter ultimas 12 entradas na secao (per D-10).

**Metodo `writeSprint(slug, entry)`** (per D-03):
- `entry: { nome: string; spEntregues: number; spPlanejados: number; cycleTimeMedio: number; entregas: string[]; bloqueios: string[] }`.
- Append na secao `## Sprints`.
- Retencao: ultimos 6 sprints (per D-10).

**Metodo `writeMonthly(slug, entry)`** (per D-04):
- `entry: { mes: string; destaques: string[]; pontosAtencao: string[]; deltasVsMesAnterior: Record<string, string> }`.
- Append na secao `## Meses`.
- Retencao: ultimos 6 meses (per D-10).

**Metodo `updateMomentoAtual(slug, momento)`** (per D-07):
- `momento: { velocity: string; cycleTime: string; codeReview: string; alertasAtivos: number }`.
- Substitui a secao `## Momento Atual` no topo do arquivo.
- Formato:
  ```
  ## Momento Atual
  - velocity: {velocity}
  - cycle time: {cycleTime}
  - code review: {codeReview}
  - alertas ativos: {alertasAtivos}
  ```
- Valores usam setas unicode: `12 SP ↑` ou `8d ↓` ou `3.2 ↔`.

**Mecanismo de secoes gerenciadas** (inspirado em ArtifactWriter):
- Usar comentarios HTML como delimitadores:
  - `<!-- METRICAS:MOMENTO_ATUAL -->` / `<!-- FIM:MOMENTO_ATUAL -->`
  - `<!-- METRICAS:ALERTAS -->` / `<!-- FIM:ALERTAS -->`
  - `<!-- METRICAS:SEMANAS -->` / `<!-- FIM:SEMANAS -->`
  - `<!-- METRICAS:SPRINTS -->` / `<!-- FIM:SPRINTS -->`
  - `<!-- METRICAS:MESES -->` / `<!-- FIM:MESES -->`
- `readSection(content, sectionKey)` retorna o conteudo entre os delimitadores.
- `replaceSection(content, sectionKey, newContent)` substitui o conteudo entre delimitadores.
- Se o arquivo nao existe, criar com template inicial contendo todas as secoes vazias.
- Se uma secao nao existe no arquivo, adiciona-la no final antes de escrever.

**Metodo privado `applyRetention(sectionContent, maxEntries)`:**
- Conta entradas pelo padrao `### ` (h3 headers dentro da secao).
- Se mais que `maxEntries`, remove as mais antigas (do topo da secao, ja que append e no final).

**Metodo privado `ensureFile(slug)`:** cria `pessoas/{slug}/metricas.md` com template se nao existe:
```markdown

<!-- METRICAS:MOMENTO_ATUAL -->

<!-- METRICAS:ALERTAS -->
## Alertas Ativos

<!-- FIM:ALERTAS -->

<!-- METRICAS:SEMANAS -->
## Semanas
<!-- FIM:SEMANAS -->

<!-- METRICAS:SPRINTS -->
## Sprints
<!-- FIM:SPRINTS -->

<!-- METRICAS:MESES -->
## Meses
<!-- FIM:MESES -->
```
