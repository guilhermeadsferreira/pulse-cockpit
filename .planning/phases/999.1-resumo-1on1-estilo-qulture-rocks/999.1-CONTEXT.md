# Phase 999.1: Resumo de 1:1 no estilo Qulture Rocks — Context

**Gathered:** 2026-03-26
**Status:** Ready for planning

<domain>
## Phase Boundary

Após a ingestão de um 1:1, gerar um resumo executivo estruturado no estilo Qulture Rocks — com o que foi discutido, compromissos assumidos (gestor e liderado separados) e próximos passos — e expô-lo diretamente no card do artefato na PersonView com botão de cópia.

**Escopo fixo:**
- Refinamento do prompt `resumo_executivo_rh` no Pass 2 (1on1-deep)
- UI: seção expandida no card do artefato com o resumo + botão "Copiar para QR"
- Sem novos campos, sem novas versões, sem mudanças no pipeline de ingestão

**Fora do escopo:**
- Versão separada "para gestor" — o cockpit interno já serve esse papel
- Resumo sob-demanda / botão de regenerar — manter fire-and-forget no Pass 2
- Nenhuma mudança na estrutura do `perfil.md` ou em outros artefatos

</domain>

<decisions>
## Implementation Decisions

### Quantas versões do resumo
- **D-01:** Uma única versão compartilhável (`resumo_executivo_rh`) — serve tanto para o liderado quanto para colar no Qulture Rocks. Não criar campos separados.
- **D-02:** O gestor já tem a visão completa (resumo da ingestão + transcrição). Não precisa de versão adicional para uso interno.

### Conteúdo do resumo (refinamento de prompt)
- **D-03:** Estrutura obrigatória do resumo:
  1. Parágrafo de abertura (2-3 frases): tópicos principais discutidos
  2. Ações do liderado (bullets com •): cada compromisso com responsável explícito
  3. Ações do gestor (bullets com •): o que o gestor se comprometeu a fazer
  4. Próximos passos (bullets com •): o que será acompanhado até o próximo 1:1
- **D-04:** Omitir do resumo: tendência emocional, insights qualitativos de carreira/PDI, análise interna, sinais sensíveis. Somente o que é adequado para o liderado ler e para registro no Qulture Rocks.
- **D-05:** Tom profissional mas direto. Formatado para colar no Qulture Rocks (campo de registro de 1:1). Sem jargões internos do app.
- **D-06:** O campo já existe como `resumo_executivo_rh` em `OneOnOneResult` e já é appendado no artefato como `## Resumo Executivo (Qulture Rocks)`. O prompt precisa de revisão para garantir qualidade — mas não é uma reescrita, é um ajuste.

### Gatilho e fluxo de geração
- **D-07:** Manter geração automática fire-and-forget no Pass 2 (1on1-deep). Zero mudanças no `IngestionPipeline` ou `ClaudeRunner`.
- **D-08:** O `ArtifactWriter.update1on1Results()` já appenda o resumo no arquivo do artefato. Nenhuma mudança necessária no Writer.

### UI — onde e como aparece
- **D-09:** No `ArtifactCard` em `PersonView.tsx`: quando o card é expandido e o artefato é do tipo `1on1`, exibir o bloco "## Resumo Executivo (Qulture Rocks)" como seção destacada **acima** do conteúdo completo do artefato.
- **D-10:** A seção destacada exibe o texto do resumo (extraído do markdown do artefato) com um botão "Copiar para QR" que usa `navigator.clipboard.writeText()`.
- **D-11:** O botão só aparece quando o artefato já foi processado pelo Pass 2 (i.e., o texto "## Resumo Executivo (Qulture Rocks)" está presente no conteúdo do artefato). Se o campo não existir, a seção não é exibida.

### Claude's Discretion
- Design visual exato da seção destacada (cor de fundo, bordas, tipografia)
- Feedback visual do botão de cópia (ex: "Copiado!" temporário)
- Como extrair exatamente a seção do markdown do artefato (regex ou parsing)

</decisions>

<specifics>
## Specific Ideas

- Destino primário do texto copiado: campo de registro de 1:1 no **Qulture Rocks**
- Referência de estrutura: "o que discutimos, o que combinamos (por responsável), próximos passos" — ata limpa, sem análise interna
- O gestor vai copiar isso para enviar ao liderado e/ou colar no QR. Precisa ser autônomo — legível sem contexto do app.
- Analogia de produto: como uma ata de reunião formal, não como um relatório interno

</specifics>

- Artefatos do tipo `1on1` são classificados pelo campo `tipo` em `ArtifactMeta`.

### Integration Points
- O conteúdo do artefato já é lido como string raw. O resumo está na seção `## Resumo Executivo (Qulture Rocks)` dentro desse conteúdo. Extração via regex sobre o `content` já carregado.
