# Phase 999.12 Plan 02: Sustentação no Weekly Report e MetricsWriter Summary

Weekly Report agora inclui seção "Sustentação da Semana" com carga por pessoa, cruzamento produtividade×suporte e persistência em metricas.md via `writeSustentacaoWeekly()`.

| Task | Description | Commit |
|------|-------------|--------|

| 2 | Integrar sustentação no WeeklyReportGenerator | 8b3e494 |

## What Was Built

### Task 1 — MetricsWriter

- Exporta `SustentacaoWeeklyEntry` com campos `semana`, `ticketsAbertos`, `breachCount`, `complianceRate7d`
- Adiciona `SUSTENTACAO_SEMANAL` ao objeto `SECTIONS` com delimitadores HTML gerenciados
- Adiciona `RETENTION.SUSTENTACAO_SEMANAL = 12` (3 meses de semanas)
- Implementa `writeSustentacaoWeekly(slug, entry)` com tabela markdown, inserção de nova linha após header separator, e retenção de 12 entradas

### Task 2 — WeeklyReportGenerator

- Importa `fetchSustentacaoForReport`, `SupportBoardSnapshot`, `SustentacaoWeeklyEntry`
- Adiciona `notifyReportProgress` com steps `init`, `sustentacao`, `build`, `write`, `done`
- Busca snapshot de sustentação (graceful — null se não configurado) com `progress('sustentacao', ..., 72)`
- Estende `buildReport()` com parâmetro opcional `sustentacao?: SupportBoardSnapshot | null`
- Adiciona seção `## Sustentação da Semana` após `## Resumo do Time` com:
  - Totais: abertos, breach count, SLA 7d
  - Carga por pessoa com alertas (>= 5 = alta, >= 3 = moderada)
  - Cruzamento inline: pessoa com >= 3 tickets E zero SP gera observação de impacto
  - Top 3 tipos de ticket
- Persiste `SustentacaoWeeklyEntry` por slug no metricas.md após geração do relatório

## Decisions Made

- **Tabela vs. blocos ###**: `writeSustentacaoWeekly` usa tabela markdown em vez do padrão `### Semana YYYY-MM-DD` dos outros métodos. Dados de suporte são tabulares; a retenção foi adaptada para contar linhas de tabela em vez de blocos `###`.
- **Seção após Resumo do Time**: Mantém hierarquia visual — sumário primeiro, detalhe de suporte depois.
- **Cruzamento condicional**: Apenas quando `count >= 3 AND storyPointsSprint === 0` — evita ruído para situações normais.
