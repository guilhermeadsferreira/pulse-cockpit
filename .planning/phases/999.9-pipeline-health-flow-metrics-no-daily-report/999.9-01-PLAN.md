**4. Criar funcao `computePipelineHealth`:**

```typescript
interface PipelinePhaseData {
  fase: string
  tasks: number
  tempoMedio: number | null
  maisAntiga: { key: string; dias: number } | null
  baseline: number | null
  status: 'ok' | 'warning' | 'critical'
}

function computePipelineHealth(
  allSprintIssues: JiraIssue[],
  activeTasks: TaskCycleInfo[],
  doneTasksPhaseTime: Map<string, number[]>,
): PipelinePhaseData[]
```

Logica:
- Definir mapa de labels de exibicao: `{ queue_dev: 'Ready for Dev', dev: 'Dev', queue_review: 'To Do CR', review: 'CR', qa: 'QA', queue_deploy: 'Ready to Deploy' }`
- Para cada fase do pipeline (na ordem: queue_dev, dev, queue_review, review, qa, queue_deploy):
  - Filtrar activeTasks por statusCategory === fase
  - tasks = count
  - tempoMedio = media de daysInStatus das tasks nesta fase (null se 0 tasks)
  - maisAntiga = task com maior daysInStatus (null se 0 tasks)
  - baseline = media dos tempos em doneTasksPhaseTime para esta fase (null se < 3 done tasks, per D-05)
  - status: comparar tempoMedio com baseline (ou threshold default se baseline null):
    - Thresholds default per D-10: Dev 5d, CR 3d, QA 3d, Ready to Deploy 2d, Ready for Dev null (sem threshold)
    - Se tempoMedio <= ref: 'ok'
    - Se tempoMedio <= 2 * ref: 'warning'
    - Se tempoMedio > 2 * ref: 'critical'
    - Se nao ha ref (Ready for Dev sem baseline): 'ok'
- Retornar array de PipelinePhaseData (omitir fases com 0 tasks — discretion do Claude)
