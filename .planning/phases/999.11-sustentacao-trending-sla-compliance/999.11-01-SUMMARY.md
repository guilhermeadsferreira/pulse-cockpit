# Phase 999.11 Plan 01: Tipos e Compliance SustentacaoBoard Summary

## One-liner

Contratos de tipos para trending/compliance de sustentação definidos em ipc.ts + cálculo de SLA compliance no SupportBoardClient com semântica null=sem dados.

## What Was Built

### Task 1: Estender tipos em ipc.ts e corrigir global.d.ts

- `SustentacaoHistoryEntry` exportada em `ipc.ts` com campos: `date`, `fetchedAt`, `ticketsAbertos`, `ticketsFechadosUltimos30d`, `breachCount`, `complianceRate7d`, `complianceRate30d`
- `SupportBoardSnapshot` estendida com `complianceRate7d: number | null`, `complianceRate30d: number | null` e `history: SustentacaoHistoryEntry[]`
- `global.d.ts` atualizado: import de `SupportBoardSnapshot` adicionado + bloco `sustentacao` com `getData`, `refresh` e `runAnalysis` tipados
- Commit: `df908b1`

### Task 2: Implementar calcularCompliance() em SupportBoardClient.ts

- Função `calcularCompliance(issues, slaThresholds, windowDias)` adicionada antes de `fetchSupportBoardMetrics`
- Retorna `null` quando `resolved.length === 0` (sem dados na janela — não exibir percentual)
- `fetchSupportBoardMetrics()` agora calcula `complianceRate7d` e `complianceRate30d` usando o `JiraIssue[]` completo em memória
- Return atualizado com os novos campos + `history: []` (preenchido pelo IPC handler no Plan 02)
- Commit: `6dd80cd`

## Decisions Made

1. **null semântica para compliance**: `calcularCompliance` retorna `null` quando não há tickets resolvidos na janela — semanticamente correto (ausência de dados != 0% compliance). A UI deve exibir "—" nesse caso.

2. **history vazio no SupportBoardClient**: O client não conhece `workspacePath` e portanto não tem acesso ao `history.json`. O array vazio é intencional — o IPC handler (Plan 02) irá substituir com dados do disco. Separação de responsabilidades correta.

3. **SustentacaoHistoryEntry sem ticketsEmBreach**: O histórico diário armazena apenas `breachCount` (número) para manter o `history.json` leve. A lista completa de tickets em breach só existe no snapshot atual.

## Deviations from Plan

None - plan executed exactly as written.

## Known Stubs

- `history: []` em `fetchSupportBoardMetrics()` — stub intencional documentado no plan. Será preenchido pelo IPC handler no Plan 02 com dados lidos de `sustentacao-history.json`.
