```typescript
// MTRC-01: Code review depth
avgCommentsPerReview: number       // media de review comments por PR revisado
firstReviewTurnaroundDias: number  // tempo medio ate primeira review (ja existe tempoMedioReviewDias — renomear internamente nao, adicionar campo dedicado com calculo identico mas semantica clara)
approvalRate: number               // % de reviews que sao 'approved' (0-100)

// MTRC-02: Collaboration
collaborationScore: number         // 0-100 composto

// MTRC-03: Test coverage
testCoverageRatio: number          // % de PRs merged que tocam arquivos de teste (0-100)
```

Implementacao em `fetchGitHubMetrics`:

1. Buscar review comments via `client.getReviewCommentsByUser(username, since)` — novo call em paralelo no Promise.all existente
2. Buscar filenames de PRs merged para detectar test files — chamar `client.getPRFilenames(config.org, pr.repo, pr.number)` para cada merged PR

**MTRC-01 calculos:**
- `avgCommentsPerReview`: total review comments / reviews.length (ou 0 se reviews.length === 0)
- `firstReviewTurnaroundDias`: ja calculado pelo existente `computeAverageReviewTime` — reutilizar o valor de `tempoMedioReviewDias`
- `approvalRate`: reviews com state === 'approved' / total reviews * 100 (arredondar com Math.round)

**MTRC-02 calculo collaborationScore:**
- co-authored commits: commits onde `commit.message` contem `Co-authored-by:` (case insensitive) → peso 30
- cross-repo activity: numero de repos distintos com atividade (commits ou PRs) / total repos configurados → peso 40
- reviews feitas: min(reviews.length / 5, 1) → peso 30
- Score = (coAuthoredRatio * 30 + crossRepoRatio * 40 + reviewRatio * 30), arredondado, clamped 0-100

**MTRC-03 calculo testCoverageRatio:**
- Para cada merged PR, chamar getPRFilenames e verificar se algum filename contem patterns de teste: `*.test.*`, `*.spec.*`, `__tests__/`, `test/`, `tests/`
- testCoverageRatio = (PRs com test files / total merged PRs) * 100, arredondado

Atualizar `EMPTY_METRICS` com os novos campos defaultando a 0.

Atualizar `summarizeGithub` em ExternalDataPass.ts para incluir os novos campos na snapshot historica — nao e arquivo deste plan, sera feito no plan 03. Aqui apenas garantir que GitHubMetrics.ts compila.
