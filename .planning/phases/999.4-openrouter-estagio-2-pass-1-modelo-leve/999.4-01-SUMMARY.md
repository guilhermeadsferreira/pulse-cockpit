# Phase 999.4 Plan 01: Pass 1 com rota híbrida OpenRouter — Summary

**One-liner:** Pass 1 do pipeline de ingestão roteado para OpenRouter com system prompt JSON e duplo fallback (erro HTTP + schema inválido) quando `useHybridModel=true`, replicando o padrão do Pass Cerimônia.

## What Was Built

### Task 1 — ClaudeRunner.ts

Adicionado `systemPrompt?: string` como 5º parâmetro opcional à função `runOpenRouterPrompt`. Quando presente, o payload enviado ao OpenRouter inclui uma mensagem de sistema antes da mensagem do usuário:

```typescript
messages: [
  ...(systemPrompt ? [{ role: 'system', content: systemPrompt }] : []),
  { role: 'user', content: prompt },
],
```

Chamadas existentes (Pass Cerimônia com 4 argumentos) continuam válidas — parâmetro é opcional.

### Task 2 — IngestionPipeline.ts

Substituído o bloco do Pass 1 em `processItem` (linha ~946) pelo padrão de roteamento híbrido com dois caminhos de fallback:

**Quando `hybridActive=true`:**
1. Chama `runOpenRouterPrompt` com `PASS1_SYSTEM_PROMPT`
2. Se falhar (HTTP/network): log warning + fallback para Claude CLI
3. Se sucesso mas schema inválido (`validateIngestionResult`): log warning + fallback para Claude CLI
4. Após fallback (ou sucesso): validação final — throw Error se Claude CLI também falhou

**Quando `hybridActive=false`:** comportamento idêntico ao anterior (Claude CLI direto).

`PASS1_SYSTEM_PROMPT` usado: `'You must respond with valid JSON only. Do not include markdown code blocks, explanations, or any text outside the JSON object.'`

## Comportamento Observável

| Cenário | Antes | Depois |
|---------|-------|--------|
| `useHybridModel=false` | Claude CLI (90s timeout) | Claude CLI (90s timeout) — inalterado |
| `useHybridModel=true`, OpenRouter OK, schema válido | Claude CLI | OpenRouter (60s timeout) |
| `useHybridModel=true`, OpenRouter HTTP error | Claude CLI | OpenRouter → warning → Claude CLI |
| `useHybridModel=true`, OpenRouter retorna schema inválido | Claude CLI (mas lançaria erro de schema) | OpenRouter → warning → Claude CLI |
| Pass 2 | Claude CLI (180s) | Claude CLI (180s) — inalterado |

## Decisões Implementadas

- **D-01 (toggle):** Mesmo `useHybridModel` existente — sem novo campo UI ou AppSettings
- **D-02 (system prompt):** Instrução JSON no campo `system` do payload OpenRouter — não modificado o prompt `buildIngestionPrompt`
- **D-03 (validação como gate):** `validateIngestionResult` atua como filtro de qualidade — schema inválido = fallback
- **D-04 (sem threshold por tamanho):** OpenRouter para todo Pass 1 quando híbrido ativo
- **D-05 (padrão de fallback):** Idêntico ao Pass Cerimônia — OpenRouter → Claude CLI → throw
